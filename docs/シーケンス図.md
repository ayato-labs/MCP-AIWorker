# Sub-cheap-McpAiAgent シーケンス図

```mermaid
sequenceDiagram
    autonumber
    actor User as 開発者 (人間)
    participant MainAI as メインAI (Claude 3.5等)
    box cyan Sub-cheap-McpAiAgent (FastMCP)
    participant MCP as MCPサーバー (Python)
    end
    database File as ローカルファイル
    participant SubLLM as サブLLM (Gemini Flash / Ollama)

    User->>MainAI: 「〇〇ファイルの20行目に機能を追加して」
    
    note over MainAI, MCP: 【トークン節約の境界線】<br/>メインAIは数行の命令を出すだけ（往復トークン激小）
    MainAI->>MCP: ツール呼び出し (指示, ファイルパス, 行番号)
    
    activate MCP
    MCP->>File: ファイルの状態確認 & コード読み込み
    activate File
    File-->>MCP: 編集対象のコード（または新規）
    deactivate File

    MCP->>MCP: プロンプト組み立て (対象コード + 指示)

    note over MCP, SubLLM: 泥臭い長文コードの読み書きは<br/>安価なサブLLM側で処理（コストほぼゼロ）
    MCP->>SubLLM: コード生成リクエスト (Gemini API / Ollama)
    activate SubLLM
    SubLLM-->>MCP: 生成された下書きコード（コードのみ）
    deactivate SubLLM

    MCP->>File: 対象の行をピンポイントで書き換え (または自動構築)
    activate File
    File-->>MCP: 書き込み完了
    deactivate File

    MCP-->>MainAI: 完了通知レスポンス (「✅ 〇行目を更新しました」)
    deactivate MCP

    
    %% ユニットテスト生成のシーケンス
    User->>MainAI: 「calculator.pyのテストを書いて」
    MainAI->>MCP: ツール呼び出し (generate_unit_tests, ソースパス)
    
    activate MCP
    MCP->>File: ソースファイル読み込み
    MCP->>SubLLM: テストコード生成リクエスト (AAAパターンのシステムプロンプト)
    activate SubLLM
    SubLLM-->>MCP: XMLブロック (<test_plan>, <draft_output>)
    deactivate SubLLM
    
    MCP->>MCP: XML解析とテストファイル生成
    MCP->>File: テストファイル書き込み
    MCP-->>MainAI: 完了通知 (生成されたTest Plan)
    deactivate MCP
```
